# Joplin 工作流说明文档

## 1. 文档目的

本文档用于总结当前 `cli-anything-joplin` 已经支持并验证通过的主要工作流，帮助开发、测试和演示人员快速了解：

- 目前已经完成了多少工作流
- 当前可以完成哪些功能
- 每个工作流分别验证什么能力
- 哪些流程适合作为独立测试、工作流测试和端到端集成测试
- 在真实 Joplin backend 下，系统是如何串起来工作的

---

## 2. 当前工作流完成情况

目前已经完成并验证通过的工作流可以按“类别”和“流程点”两种方式理解。

### 2.1 按类别统计

当前工作流体系共分为 **7 类**：

1. 独立命令工作流
2. 笔记生命周期工作流
3. 标签工作流
4. 搜索工作流
5. 同步工作流
6. 导出工作流
7. 项目保存 / 完整端到端工作流

### 2.2 按可验证流程点统计

如果把每个可单独验证的命令或流程也算进去，当前大约有 **18 个流程点**：

#### 独立命令类（11 个）
- `project new`
- `project info`
- `project status`
- `project save`
- `session status`
- `notebooks list`
- `notes list`
- `config get`
- `config list`
- `session undo`
- `session redo`

#### 短工作流类（6 个）
- 笔记生命周期工作流
- 标签工作流
- 搜索工作流
- 同步工作流
- 导出工作流
- 项目保存工作流

#### 端到端集成类（1 个）
- 完整项目 roundtrip 工作流

---

## 3. 当前可以完成什么功能

### 3.1 项目管理

`cli-anything-joplin` 现在已经可以完成项目的初始化、查看、保存和状态检查。

**能力包括：**
- 新建项目文件
- 打开已有项目
- 查看项目信息
- 查看项目状态
- 保存项目状态
- 检查当前 session 是否已加载项目

---

### 3.2 Notebook 管理

当前可以对 notebook 做基础管理和选择操作。

**能力包括：**
- 列出 notebook
- 创建 notebook
- 切换当前 notebook

---

### 3.3 Note 管理

当前已经具备完整的 note 生命周期操作能力。

**能力包括：**
- 列出 note
- 创建 note
- 修改 note 字段
- 获取 note 内容
- 删除 note

---

### 3.4 Tag 管理

当前已经可以对 note 进行标签操作，并读取标签关联信息。

**能力包括：**
- 列出 tag
- 给 note 添加 tag
- 删除 tag
- 查看某个 note 绑定了哪些 tag

---

### 3.5 搜索能力

当前可以执行搜索命令，用于定位 note 内容。

**能力包括：**
- 按关键字搜索 note
- 在完整工作流里验证搜索链路可用

---

### 3.6 同步能力

当前可以执行同步命令，用于验证 backend 的同步入口是否正常。

**能力包括：**
- 执行同步
- 在真实 backend 场景下验证同步命令可运行

---

### 3.7 导入 / 导出能力

当前已经完成导出链路，并补充了导入链路验证。

**能力包括：**
- 导出为 `jex`
- 导出为 `md`
- 导入外部 markdown 文件

---

### 3.8 配置与会话能力

当前已经具备配置读取和 session 管理能力。

**能力包括：**
- 读取配置项
- 列出配置
- 查看 session 状态
- 验证 undo / redo 的边界行为

---

## 4. 当前测试层级与工作流对应关系

### `TestCLISubprocess`
- 负责 CLI 基础行为和命令入口
- 对应独立命令类验证

### `TestBackendCommands`
- 负责单个 backend 命令的独立验证
- 对应独立命令类验证

### `TestBackendWorkflows`
- 负责短链路工作流验证
- 对应笔记、标签、搜索、同步、导出、保存等流程

### `TestBackendIntegration`
- 负责完整端到端演示流程
- 对应完整 roundtrip 工作流

---

## 5. 总结

目前 `cli-anything-joplin` 已经完成 **7 类主要工作流**，并且从更细的视角看，已经覆盖了大约 **18 个可验证流程点**，可以覆盖：

- 项目管理
- Notebook 管理
- Note 管理
- Tag 管理
- 搜索
- 同步
- 导入 / 导出
- 配置读取
- Session 状态管理
- 完整端到端 roundtrip 演示

整体上，这套工作流已经足够支持：

- 独立命令测试
- 短工作流测试
- 端到端集成测试
- agent 演示
- 真实 backend 回归验证

---

## 6. 结论

当前的工作流体系已经形成了清晰的分层，并且后续可以继续按需要扩展更多工作流点。

当前的工作流体系已经形成了清晰的分层：

- 适合继续扩展
- 适合做展示
- 适合做回归

- **单点命令** 保证基础能力正确
- **短工作流** 保证真实操作路径可用
- **完整集成流** 保证 agent 演示和真实 backend roundtrip 可跑通

这套结构已经可以作为当前项目的主要工作流说明基线。
